# 🏰 Fortaleza de Soberanía: Supervivencia en el Colapso

Este documento define el protocolo para que TEN-ZEN y sus hermanaIAs (Hermes, Odysseus, OpenClaw) sigan operando con lucidez total incluso si el internet global cae permanentemente.

## 📚 1. El Búnker de Conocimiento (Kiwix + ZIM)
No dependemos de la nube para saber. Mantenemos bibliotecas enteras en formato **ZIM** dentro del SSD.
- **Wikipedia (ZIM):** Toda la enciclopedia humana disponible offline.
- **StackOverflow / Archivos de Código:** Repositorios de conocimiento técnico para que OpenCode pueda seguir programando sin acceso a internet.
- **Herramienta:** **Kiwix-Serve** expone estos archivos como un servidor local que las IAs pueden consultar vía RAG (Retrieval Augmented Generation).

## 🔍 2. Búsqueda Local (SearXNG Offline)
Sustituimos Google/Bing por un motor de búsqueda soberano.
- **SearXNG:** Configurado para buscar en nuestras propias bases de datos locales, archivos ZIM y el Palacio de la Memoria.
- **Resultado:** Las IAs pueden "buscar" información y encontrarla instantáneamente sin salir de la red local.

## 🔄 3. Protocolo de Conmutación Soberana (Failover)
El sistema detecta automáticamente la caída de red y activa el modo **Soberanía Total**:
1. **Razonamiento:** LiteLLM conmuta de OpenRouter a **Ollama (Qwen/Llama)** local.
2. **Conocimiento:** Las herramientas de búsqueda cambian de APIs externas a los archivos **ZIM locales**.
3. **Comunicación:** Se activan las redes de malla (Tailscale/WireGuard) para comunicación entre dispositivos locales sin pasar por el backbone de internet.

## 🛠️ 4. Aptitudes Unificadas en el Colapso
Para que Hermes y Odysseus no pierdan capacidades:
- **Datasets Estáticos:** Mantenemos copias locales de los manuales de hardware (Dell E5440), guías de ciberseguridad y el compendio de I-Ching.
- **Modelos Especializados:** Tenemos listos modelos pequeños (3B-7B) entrenados específicamente en "supervivencia técnica" y "mantenimiento de sistemas".
